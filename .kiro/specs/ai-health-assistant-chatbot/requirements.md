# Requirements Document

## Introduction

The AI Health Assistant Chatbot is a frontend-only conversational interface for the HealMate healthcare application. It provides patients with instant access to medication reminders, adherence insights, and basic health guidance through a modern chat interface. The chatbot uses mock AI responses for demonstration purposes, with a structure prepared for future backend API integration.

## Glossary

- **Chatbot**: The AI Health Assistant conversational interface component
- **Chat_Window**: The modal panel displaying the conversation interface
- **Chat_Button**: The floating action button that opens the Chat_Window
- **User_Message**: A message sent by the patient user
- **Bot_Message**: A response generated by the Chatbot
- **Message_Input**: The text input field where users type messages
- **Quick_Action_Button**: Pre-defined question buttons for common queries
- **Mock_Response_Engine**: The client-side logic that generates responses based on keyword matching
- **Chat_Service**: The service layer that handles message processing and future API integration

## Requirements

### Requirement 1: Chatbot Access

**User Story:** As a patient, I want to access the AI health assistant from any page, so that I can get help whenever I need it.

#### Acceptance Criteria

1. THE Chat_Button SHALL be displayed as a floating button in the bottom-right corner of all patient pages
2. WHEN the Chat_Button is clicked, THE Chatbot SHALL open the Chat_Window with a slide-up animation
3. WHEN the Chat_Window is open and the Chat_Button is clicked, THE Chatbot SHALL close the Chat_Window
4. THE Chat_Button SHALL display a chat or AI assistant icon
5. THE Chat_Window SHALL overlay the main content without blocking critical UI elements

### Requirement 2: Chat Interface Display

**User Story:** As a patient, I want a clean and familiar chat interface, so that I can easily communicate with the assistant.

#### Acceptance Criteria

1. THE Chat_Window SHALL display a header containing the title "HealMate AI Assistant" and an online status indicator
2. THE Chat_Window SHALL display User_Messages aligned to the right side with distinct styling
3. THE Chat_Window SHALL display Bot_Messages aligned to the left side with distinct styling
4. THE Chat_Window SHALL use rounded chat bubbles for all messages
5. THE Chat_Window SHALL use healthcare-themed colors (green or blue tones) for the interface
6. THE Chat_Window SHALL display messages in chronological order with newest messages at the bottom
7. THE Chat_Window SHALL automatically scroll to show the most recent message when a new message is added

### Requirement 3: Message Sending

**User Story:** As a patient, I want to send messages to the assistant, so that I can ask questions and get help.

#### Acceptance Criteria

1. THE Message_Input SHALL accept text input from the user
2. WHEN the user clicks the send button, THE Chatbot SHALL add the User_Message to the Chat_Window
3. WHEN the user presses the Enter key in the Message_Input, THE Chatbot SHALL add the User_Message to the Chat_Window
4. WHEN a User_Message is sent, THE Chatbot SHALL clear the Message_Input field
5. WHEN a User_Message is sent, THE Chatbot SHALL display the message with a fade-in animation
6. THE Message_Input SHALL display a send button icon
7. IF the Message_Input is empty, THEN THE Chatbot SHALL disable the send button

### Requirement 4: Mock AI Response Generation

**User Story:** As a patient, I want to receive instant responses from the assistant, so that I can get immediate help with my health queries.

#### Acceptance Criteria

1. WHEN a User_Message is sent, THE Mock_Response_Engine SHALL generate a Bot_Message within 500 milliseconds
2. IF the User_Message contains "missed dose" or "missed medicine", THEN THE Mock_Response_Engine SHALL return "It seems you missed a dose. Please take it if safe or consult your doctor."
3. IF the User_Message contains "medicine" or "medication", THEN THE Mock_Response_Engine SHALL return "Make sure to follow your prescribed schedule for best results."
4. IF the User_Message contains "adherence" or "compliance", THEN THE Mock_Response_Engine SHALL return "Your adherence is important for your health. Check your dashboard for detailed insights."
5. IF the User_Message contains "schedule" or "reminder", THEN THE Mock_Response_Engine SHALL return "I can help you stay on track with your medication schedule. Check your medication list for timing details."
6. IF the User_Message does not match any keyword pattern, THEN THE Mock_Response_Engine SHALL return "I'm here to help with your medication and health queries."
7. WHEN the Mock_Response_Engine is generating a response, THE Chatbot SHALL display a typing indicator animation

### Requirement 5: Quick Action Buttons

**User Story:** As a patient, I want to see suggested questions, so that I can quickly access common information without typing.

#### Acceptance Criteria

1. WHEN the Chat_Window is opened for the first time, THE Chatbot SHALL display Quick_Action_Buttons below the welcome message
2. THE Chatbot SHALL display a Quick_Action_Button with text "Did I miss my medicine?"
3. THE Chatbot SHALL display a Quick_Action_Button with text "What is my adherence?"
4. THE Chatbot SHALL display a Quick_Action_Button with text "Remind me my schedule"
5. WHEN a Quick_Action_Button is clicked, THE Chatbot SHALL send the button text as a User_Message
6. WHEN a Quick_Action_Button is clicked, THE Chatbot SHALL hide all Quick_Action_Buttons

### Requirement 6: Chat Animations

**User Story:** As a patient, I want smooth and pleasant animations, so that the chat experience feels polished and professional.

#### Acceptance Criteria

1. WHEN the Chat_Window opens, THE Chatbot SHALL animate the window with a slide-up transition over 300 milliseconds
2. WHEN a message is added, THE Chatbot SHALL animate the message with a fade-in transition over 200 milliseconds
3. WHEN the Mock_Response_Engine is processing, THE Chatbot SHALL display an animated typing indicator with three pulsing dots
4. WHEN the Chat_Window closes, THE Chatbot SHALL animate the window with a slide-down transition over 300 milliseconds

### Requirement 7: Backend API Preparation

**User Story:** As a developer, I want the chatbot structured for future API integration, so that we can easily connect to a real AI backend later.

#### Acceptance Criteria

1. THE Chat_Service SHALL define a message format structure with fields for message text, timestamp, and sender type
2. THE Chat_Service SHALL include a placeholder function for POST requests to "/api/chat" endpoint
3. THE Chat_Service SHALL structure request payloads as JSON objects with a "message" field containing user input
4. THE Chat_Service SHALL include error handling structure for future API failures
5. THE Mock_Response_Engine SHALL be implemented as a separate, replaceable function within Chat_Service

### Requirement 8: Component Structure

**User Story:** As a developer, I want a clean component architecture, so that the code is maintainable and follows React best practices.

#### Acceptance Criteria

1. THE Chatbot SHALL be implemented in a file at "frontend/src/components/Chatbot.jsx"
2. THE Chatbot SHALL use a ChatMessage component defined in "frontend/src/components/ChatMessage.jsx"
3. THE Chat_Service SHALL be implemented in a file at "frontend/src/services/chatService.js"
4. THE Chatbot component SHALL manage chat state using React hooks
5. THE ChatMessage component SHALL accept props for message text, sender type, and timestamp
6. THE Chatbot SHALL be responsive and adapt to mobile screen sizes

### Requirement 9: Initial Welcome Message

**User Story:** As a patient, I want to see a welcome message when I open the chat, so that I understand what the assistant can help me with.

#### Acceptance Criteria

1. WHEN the Chat_Window is opened for the first time in a session, THE Chatbot SHALL display a welcome Bot_Message
2. THE welcome Bot_Message SHALL contain text "Hello! I'm your HealMate AI Assistant. I can help you with medication reminders, adherence insights, and health guidance. How can I assist you today?"
3. THE welcome Bot_Message SHALL appear before any Quick_Action_Buttons

### Requirement 10: Chat Persistence

**User Story:** As a patient, I want my chat history to remain visible during my session, so that I can reference previous responses.

#### Acceptance Criteria

1. WHILE the Chat_Window is open, THE Chatbot SHALL maintain all messages in the conversation history
2. WHEN the Chat_Window is closed and reopened, THE Chatbot SHALL display all messages from the current session
3. WHEN the user refreshes the page or logs out, THE Chatbot SHALL clear the conversation history
